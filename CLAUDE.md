## Commands

```bash
make generate         # regenerate internal/client/client.gen.go from api/komodor-clean.yaml
make build            # produces ./bin/komodor
make lint             # golangci-lint run ./...
make test             # unit tests (no network required)
make test-integration # integration tests — requires KOMODOR_API_KEY env var
go test ./internal/cmd/ -run TestServicesSearch  # run a single test by name
```

## Architecture

```
cmd/komodor/main.go              # entrypoint — delegates to internal/cmd.Execute()
internal/
  auth/auth.go                   # API key resolution (flag > env > config file)
  config/config.go               # ~/.config/komodor/config.yaml read/write
  output/output.go               # Formatter interface + table/json/yaml/csv implementations
  client/client.gen.go           # generated HTTP client — DO NOT EDIT (22k lines)
  cmd/
    root.go                      # global flags, PersistentPreRunE auth middleware, context injection
    complete.go                  # dynamic shell completion functions (live API calls)
    helpers.go                   # shared utilities: apiError, strPtr, joinStrings, etc.
    testhelpers_test.go          # shared test infrastructure: mockClient, captureFormatter, newTestRoot
    *.go                         # one file per resource group (services, clusters, users, etc.)
api/
  komodor.yaml                   # original vendored OpenAPI spec
  komodor-clean.yaml             # cleaned spec used for codegen (x-go-type stripped, name conflicts resolved)
  oapi-codegen.yaml              # codegen config
```

### Request flow

`PersistentPreRunE` in `root.go` runs before every authenticated command:
1. Loads `~/.config/komodor/config.yaml`
2. Resolves API key: `--api-key` flag > `KOMODOR_API_KEY` env > config file
3. Creates `*client.ClientWithResponses` with the `x-api-key` header injected
4. Creates a `output.Formatter` based on `--output` flag (`table`|`json`|`yaml`|`csv`)
5. Stores both in cobra's context — retrieved in commands via `clientFromCtx` / `formatterFromCtx`

Auth-exempt commands (skip `PersistentPreRunE`): `auth set-key`, `auth show`, `completion`, `help`, `__complete`, `__completeNoDesc`.

## Key Patterns

### Adding a new command

Each resource group lives in its own file (`internal/cmd/<resource>.go`). The pattern is:
1. Define a parent `newXxxCmd()` that groups subcommands
2. Each subcommand calls `clientFromCtx(cmd)` and `formatterFromCtx(cmd)` to get dependencies
3. For list outputs, implement `output.TableData` (`Headers() []string`, `Rows() [][]string`) on a local struct and pass it to `formatter.Print()`
4. Register the parent in `NewRootCmd()` in `root.go`

The `client.ClientWithResponsesInterface` (generated interface) is what commands depend on, not the concrete type. This is what enables mock injection in tests.

### Testing

Tests use `newTestRoot` from `testhelpers_test.go`, which replaces `PersistentPreRunE` to inject a `*mockClient` and `captureFormatter` — no real API calls, no config files needed. `mockClient` embeds `client.ClientWithResponsesInterface` and panics on any unimplemented method, so tests fail loudly on unexpected calls.

```go
mock := &mockClient{
    postServicesSearch: func(...) (*client.PostApiV2ServicesSearchResponse, error) { ... },
}
var buf bytes.Buffer
root := newTestRoot(t, mock, &buf)
out, err := runCmd(root, &buf, "services", "search")
```

### Generated client

`internal/client/client.gen.go` is generated by `oapi-codegen v2.4.1` from `api/komodor-clean.yaml`. Never edit it directly — always regenerate via `make generate`. The clean spec differs from the original because it has `x-go-type`/`x-go-type-import` extensions stripped and duplicate typenames resolved.

### Shell completion

`complete.go` provides live API completions for `--cluster`, `--role`, `--policy`, and `--service-id` flags. Completion functions build their own client from env/config only (flags are not parsed during completion). Any failure silently returns empty — completion must never break the shell.
